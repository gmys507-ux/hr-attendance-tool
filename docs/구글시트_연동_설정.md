# 구글시트 연동 설정 가이드 (1회만)

매월 근태정리 결과를 **구성원 공유용 구글시트에 새 탭으로 자동 기록**하기 위한 1회성 설정입니다.
서비스 계정(로봇 계정) 방식이라, 한 번 설정하면 이후엔 도구가 자동으로 탭을 만들고 채웁니다.

---

## A. 구글 클라우드에서 서비스 계정 만들기

> 구글 클라우드 콘솔 = 브라우저로 접속하는 웹사이트. 별도 설치 없음.
> 모든 단계는 회사 구글 계정 `younsung@2359.co.kr` 로그인 상태에서 진행.

### A-1. 프로젝트 만들기
- 딥링크: https://console.cloud.google.com/projectcreate
- **프로젝트 이름**: `HR-근태자동화` 입력 → **만들기**
- 생성되면 상단 파란 바의 프로젝트 선택기에서 이 프로젝트가 선택됐는지 확인

### A-2. Google Sheets API 사용 설정
- 딥링크: https://console.cloud.google.com/apis/library/sheets.googleapis.com
- (상단에 방금 만든 프로젝트가 선택돼 있는지 확인) → 파란 **사용** 버튼 클릭

### A-3. 서비스 계정 만들기
- 딥링크: https://console.cloud.google.com/iam-admin/serviceaccounts
- 상단 **+ 서비스 계정 만들기**
  - 서비스 계정 이름: `attendance-bot` → **만들기 및 계속**
  - 역할 선택: 비워둬도 됨 → **계속** → **완료**

### A-4. JSON 키 다운로드 (가장 중요)
- A-3에서 만들어진 서비스 계정을 **클릭**해서 들어감
- 상단 탭 중 **키(KEYS)** 탭 클릭
- **키 추가 → 새 키 만들기 → JSON 선택 → 만들기**
- → `hr-근태자동화-xxxxx.json` 파일이 자동 다운로드됨 (보통 `다운로드` 폴더)
- ⚠️ 이 파일이 비밀번호 역할입니다. 외부 공유·업로드 금지.

---

## B. 대상 구글시트를 서비스 계정에 공유

1. 받은 JSON 파일을 메모장으로 열어 `"client_email"` 값을 복사
   - 예: `attendance-bot@hr-근태자동화.iam.gserviceaccount.com`
2. 대상 구글시트 열기:
   https://docs.google.com/spreadsheets/d/1OHDcoW9I9wTZg3Zkbp3b8IwI0CXlpwRFwiLu6H6iZQs
3. 우측 상단 **공유** → 복사한 서비스계정 이메일 붙여넣기 → 권한 **편집자** → 전송

> ⚠️ 이 단계를 빼먹으면 "권한 없음" 오류가 납니다. 반드시 편집자로 공유하세요.

---

## C. 도구에 키 등록

1. `시작.bat`으로 도구 실행 → **📊 근태정리 생성** 페이지
2. 근태정리를 한 번 생성하면 하단에 **"구글시트 공유"** 섹션이 나타납니다.
3. **서비스계정 JSON 키 업로드** 칸에 A에서 받은 JSON 파일 업로드 (한 번만)
   - 저장 위치: `data/gcp_service_account.json`
4. 화면에 서비스계정 이메일이 표시되면 정상.

---

## D. 매월 사용 (설정 후)

1. 근태정리 생성 (원본 3종 업로드 → 생성)
2. 하단 **구글시트 공유** 섹션에서:
   - 새 탭 이름 확인 (기본값: 해당 월, 예 `5월`)
   - **📤 구글시트에 탭 추가** 클릭
3. 끝 → 구성원들이 구글시트에서 새 탭 확인 가능

- **대상 인원**: 연봉제 제외(비연봉제만)
- **포맷**: 기존 구성원시트와 동일 (No·이름·소정·4주차·초과·야간·출처, 계약종류 없음)
- 같은 이름 탭이 이미 있으면 기본은 거부. "덮어쓰기" 체크 시 갱신.
