# 📝 참가자 워크시트 — AI로 내 업무 자동화하기

> **오늘의 흐름**: "그냥 AI한테 만들어줘"가 아니라 → ① 내 업무를 그리고 → ② 자동화할 곳을 찾고 → ③ 명세(PRD)를 써서 → ④ AI에게 제대로 시키고 → ⑤ 검증하기.
> 아래 5개 워크시트를 순서대로 채워보세요. 각 시트엔 **HR 근태 + 영업/마케팅/재무 완성 예시**가 있으니 막힐 때 자기 부서와 비슷한 걸 참고하세요.

## 워크시트 1. 워크플로우 매핑 캔버스

> 목적: 내가 반복하는 업무 하나를 "입력 → 처리 → 출력"으로 쪼갠다.
> 소요: 5~10분. 잘 쓰는 게 아니라 **머릿속 일을 눈에 보이게 꺼내는** 게 목적.

### 1-A. 업무를 한 줄로
- 자동화하고 싶은 반복 업무: `________________________________`
- 빈도 (매일/매주/매월/분기): `__________`
- 1회 소요 시간: `__________`

### 1-B. 단계별 쪼개기 (빈 템플릿)

| # | 단계 (내가 하는 일) | 구분(입력/처리/출력) | 사용 도구 | 1회 소요 | 빈도 | 고통도(1~5) |
|---|---|---|---|---|---|---|
| 1 |  |  |  |  |  |  |
| 2 |  |  |  |  |  |  |
| 3 |  |  |  |  |  |  |
| 4 |  |  |  |  |  |  |
| 5 |  |  |  |  |  |  |
| 6 |  |  |  |  |  |  |

> 고통도 = 귀찮음 + 실수 잘 남 + 시간 잡아먹음. 직감으로 1(편함)~5(매번 괴로움).

### 1-C. 완성 예시

**① HR — 월간 근태정리** (빈도: 매월 / 1회: 4~6시간)

| # | 단계 | 구분 | 도구 | 소요 | 빈도 | 고통 |
|---|---|---|---|---|---|---|
| 1 | 시스템 2곳(원티드·FLEX) 엑셀 다운로드 | 입력 | 웹 2개 | 20분 | 매월 | 2 |
| 2 | 두 엑셀을 직원 기준 수작업 병합 | 처리 | 엑셀(복붙·vlookup) | 60분 | 매월 | 5 |
| 3 | 43명 소정 대비 초과/야간 계산 | 처리 | 엑셀 수식 | 90분 | 매월 | 4 |
| 4 | 엣지케이스(동명이인·전환자) 수기 처리 | 처리(판단) | 머리+엑셀 | 60분 | 매월 | 5 |
| 5 | 숫자 검증 | 처리 | 눈+엑셀 | 40분 | 매월 | 4 |
| 6 | 보고 양식 정리·공유 | 출력 | 엑셀/메일 | 30분 | 매월 | 2 |

**② 영업 — 주간 매출 취합** (빈도: 매주 / 1회: 약 1.5시간)

| # | 단계 | 구분 | 도구 | 소요 | 빈도 | 고통 |
|---|---|---|---|---|---|---|
| 1 | 3개 지점에서 매출 엑셀 수합(메일) | 입력 | 메일/엑셀 | 15분 | 매주 | 3 |
| 2 | 지점마다 다른 양식을 하나로 통일 | 처리 | 엑셀(복붙) | 40분 | 매주 | 5 |
| 3 | 품목별·지점별 합계 계산 | 처리 | 엑셀 수식 | 25분 | 매주 | 3 |
| 4 | 전주 대비 증감 정리 | 처리 | 엑셀 | 15분 | 매주 | 3 |
| 5 | 보고용 표·메일 작성 | 출력 | 엑셀/메일 | 15분 | 매주 | 2 |

**③ 마케팅 — 캠페인 성과 리포트** (빈도: 매주/매월 / 1회: 약 2시간)

| # | 단계 | 구분 | 도구 | 소요 | 빈도 | 고통 |
|---|---|---|---|---|---|---|
| 1 | 채널별(네이버·메타·구글) 광고 데이터 내려받기 | 입력 | 광고 콘솔 3곳 | 30분 | 매주 | 4 |
| 2 | 채널마다 다른 컬럼을 공통 양식으로 정리 | 처리 | 엑셀 | 45분 | 매주 | 5 |
| 3 | 비용·전환·ROAS 계산 | 처리 | 엑셀 수식 | 30분 | 매주 | 4 |
| 4 | 캠페인별 요약 + 인사이트 코멘트 | 처리(판단) | 머리+엑셀 | 30분 | 매주 | 3 |
| 5 | 리포트 양식으로 정리·공유 | 출력 | 엑셀/슬라이드 | 15분 | 매주 | 2 |

**④ 재무 — 부서별 경비 정산 취합** (빈도: 매월 / 1회: 약 3시간)

| # | 단계 | 구분 | 도구 | 소요 | 빈도 | 고통 |
|---|---|---|---|---|---|---|
| 1 | 부서별 경비 엑셀 수합 | 입력 | 메일/공유드라이브 | 30분 | 매월 | 3 |
| 2 | 계정과목 분류·표기 통일 | 처리 | 엑셀 | 50분 | 매월 | 5 |
| 3 | 부서·계정별 합산 | 처리 | 엑셀 수식 | 40분 | 매월 | 3 |
| 4 | 한도 초과·증빙 누락 점검 | 처리(판단) | 눈+엑셀 | 40분 | 매월 | 5 |
| 5 | 결산 양식 정리·보고 | 출력 | 엑셀 | 20분 | 매월 | 2 |

### 작성 팁
- 단계는 4~8개면 충분. 너무 잘게 쪼개지 마세요.
- "처리" 안에 **사람 판단이 꼭 필요한 단계**는 (판단)이라 적어두세요(다음 시트에서 씀).
- 시간은 "체감"으로 적어도 OK. 정확도보다 비교가 중요.

## 워크시트 2. 자동화 기회 발굴 시트

> 목적: 워크시트1 각 단계를 점수로 매겨 **어디부터 AI에게 맡길지** 우선순위 도출.
> 원칙: **반복적·규칙적인 일 = AI / 판단·정책·책임 = 사람.**

### 2-A. 평가축 (각 1~5점, 5에 가까울수록 자동화 적합)

| 평가축 | 뜻 | 1점 | 5점 |
|---|---|---|---|
| 반복빈도 | 얼마나 자주 반복 | 가끔 | 매번 똑같이 |
| 규칙성 | 규칙으로 설명 가능? | 그때그때 다름 | 규칙 명확 |
| 소요시간 | 시간 많이 먹나 | 금방 | 오래 |
| 실수위험 | 손으로 하면 틀리나 | 거의 안 틀림 | 자주 실수 |
| AI적합도 | 텍스트·표·계산 위주? | 감각/대면/협상 | 데이터·계산·양식 |

**합계 해석**: 20~25 = 자동화 1순위 / 14~19 = 2순위(부분) / 13↓ = 지금은 손으로(또는 사람 영역)

### 2-B. 점수표 (빈 템플릿)

| 단계 | 반복 | 규칙 | 시간 | 실수 | AI적합 | 합계 | 사람판단? |
|---|:--:|:--:|:--:|:--:|:--:|:--:|:--:|
|  |  |  |  |  |  |  | □ |
|  |  |  |  |  |  |  | □ |
|  |  |  |  |  |  |  | □ |

### 2-C. 사람이 해야 하는 판단 분리
- 이 업무에서 사람이 꼭 결정해야 하는 것: `____________________`
- AI에게 맡길 범위(어디까지): `____________________`

### 2-D. 완성 예시 — HR 근태정리

| 단계 | 반복 | 규칙 | 시간 | 실수 | AI적합 | 합계 | 사람판단? |
|---|:--:|:--:|:--:|:--:|:--:|:--:|:--:|
| 2. 두 엑셀 병합 | 5 | 5 | 4 | 4 | 5 | **23** | □ |
| 3. 초과/야간 계산 | 5 | 5 | 5 | 5 | 5 | **25** | □ |
| 5. 검증 | 5 | 4 | 3 | 5 | 5 | **22** | □ |
| 4. 엣지케이스 처리 | 5 | 2 | 4 | 5 | 3 | 19 | ☑ (정책 판단) |

해석: 병합·계산·검증(23~25)은 **자동화 1순위**. "새 케이스를 어떤 규정으로 처리할지" 결정은 사람, "정해진 규칙대로 적용·계산"은 AI.
- 사람 결정: 모호한 근태 케이스 규정 적용, 최종 보고 책임
- AI 범위: 병합 → 규칙 기반 계산 → 검증표 생성 → 양식화 (판단 필요 행은 "확인필요" 표시)

> 💡 영업/마케팅/재무도 패턴 동일: "통일·합산·계산·양식화"는 AI 1순위, "인사이트 해석·한도 예외 판단·보고 책임"은 사람.

### 작성 팁
- 점수는 **단계끼리 비교**가 목적. 절대점수 집착 X.
- 규칙성이 2점 이하면 → 자동화보다 **규칙부터 정리**하라는 신호.
- 사람 판단 체크된 단계는 통째로 빼지 말고 **역할을 쪼개**세요(사람:판단 / AI:계산).

## 워크시트 3. 미니 PRD 템플릿 (1페이지)

> PRD = 요구사항 명세 = AI에게 줄 **주문서**. 빈칸을 채우면 명세가 완성됩니다. 코딩 지식 불필요.
> (단독 배포용 + 부서별 완성 예시 풀버전은 **`05_예제_미니PRD_템플릿.md`** 참고)

```
[미니 PRD]   작성자:______   날짜:______

1. 문제 / 목적
   - 지금 무엇이 불편한가:
   - 이루고 싶은 것(한 문장):
2. 사용자 (누가 쓰나 / 언제)
3. 입력 (무엇이 들어오나 — 파일/형식/출처)
4. 출력 (결과물 모양 — 시트/컬럼/형식)
5. 핵심 규칙·계산 (AI가 따라야 할 규칙)
6. 엣지케이스 (예외 상황 — 이런 경우엔 이렇게)
7. 하지 않을 것 (Non-goals)
8. 성공 기준 (이러면 성공 — 무엇과 대조해 검증?)
```

### 완성 예시 (요약) — HR 근태정리
- ①문제: 시스템 2곳 데이터 수작업 병합/계산 4~6시간 → ②사용자: HR 담당, 매월 → ③입력: 원티드·FLEX 엑셀 → ④출력: 5시트 엑셀 + 구글시트 공유 → ⑤규칙: 사번 기준 병합, 소정 대비 초과/야간 계산 → ⑥예외: 동명이인·전환자·소정 불일치는 "확인필요" → ⑦Non-goal: 급여 계산 안 함 → ⑧성공: 수기 결과와 100% 일치 + 30분 이내

### 작성 팁
- **5번(규칙)·6번(예외)이 심장.** 구체적일수록 AI 결과가 정확.
- **7번(하지 않을 것) 꼭 채우기** — 범위를 줄여야 첫 결과가 빨리 나옴.
- 모르는 칸은 "모름/확인필요"라 적어도 됨(그게 AI에게 물어볼 거리).

## 워크시트 4. "AI에게 잘 시키는 법" 체크리스트

> 다 쓴 PRD를 **AI가 알아듣는 지시**로 바꾸는 5가지.

```
□ 1. 맥락부터 준다  — 나는 누구, 무슨 업무, "PRD 첨부할게"
□ 2. 한 번에 다 시키지 않는다  — 병합 → 계산 → 검증 → 양식화 단계로, 중간 확인
□ 3. PRD를 그대로 붙인다  — "알아서"가 아니라 입력/출력/규칙/예외를 그대로
□ 4. 산출물을 검증한다 (★최重要)  — 손으로 아는 정답과 대조, "검증 시트도 만들어줘"
□ 5. 반복 개선한다  — 틀린 부분만 콕 집어 "이 케이스는 이렇게 바꿔줘"
```

### 나쁜 vs 좋은 프롬프트 (HR 사례)

❌ **나쁜 예**: "근태 정리해줘"
→ 어떤 데이터인지, 무슨 계산인지, 결과 형태가 뭔지 하나도 없음 → AI가 추측 → 엉뚱한 결과

✅ **좋은 예** (PRD를 말로 풀어쓴 것):
- 맥락: "나는 HR 근태 담당. 매월 직원 43명 근태 정리. 아래 PRD 첨부할게."
- 입력: "원티드·FLEX 엑셀 2개를 올릴 거야. 컬럼명이 서로 달라(성명 vs name). 사번 기준 병합."
- 규칙: "소정 대비 초과/야간 구분 계산. 반올림은 PRD 규칙대로."
- 단계: "먼저 병합 결과만 보여줘. 확인하면 다음 계산으로."
- 검증: "끝나면 '확인필요/이상치' 시트도 따로 만들어줘."
- 출력: "최종은 5시트 엑셀. 급여 계산은 하지 마(범위 밖)."

### 작성 팁
- "좋은 프롬프트 = 짧은 PRD를 말로 풀어쓴 것."
- AI가 헤매면 더 자세히가 아니라 **더 작게 쪼개서** 시키기.
- 4번(검증)을 건너뛰면 자동화가 아니라 **사고**입니다.

## 워크시트 5. (선택) 세션 후 과제

> 오늘 흐름을 내 진짜 업무에 적용. 다음 모임 전까지.

```
□ 1. 내 반복 업무 1개 선정 → 워크플로우 매핑 (워크시트1)
□ 2. 자동화 점수표 작성 → 1순위 단계 선정 (워크시트2)
□ 3. 그 단계로 미니 PRD 1장 완성 (워크시트3 / 05번 템플릿)
□ 4. Claude Code/Cursor에 PRD 붙여 "1단계(맥락+첫 처리)"만 시켜보기
□ 5. 나온 결과를 손계산/기존 결과와 1건 이상 대조(검증)
□ 6. 막힌 점 1개 + 잘된 점 1개 메모해오기
```

**제출/공유 칸**
- 내가 고른 업무: `____________________`
- 자동화 1순위 단계: `____________________`
- PRD에서 가장 막힌 칸: `____________________`
- AI 결과를 검증한 방법·결과: `____________________`

### 작성 팁
- 처음부터 완성 앱 노리지 말 것. **"PRD 1장 + 첫 단계 시켜보기"까지가 목표.**
- 실패해도 OK. "어디서 막혔는지"가 다음 세션 최고의 재료.

### (강사용 메모 · 참가자 배포 X)
- 워크시트 작성 직전 **해당 부서와 비슷한 완성 예시를 먼저** 보여주면 진입장벽이 확 낮아짐.
- 가장 강조할 두 지점: 워크시트2의 **"사람 판단 영역 분리"**, 워크시트4의 **"검증"**. 비개발자가 가장 자주 놓치는 부분.
