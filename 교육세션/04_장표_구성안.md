# 🖼️ 장표(슬라이드) 구성안 + 제작 방식 가이드

> 1.5~2시간 워크숍용 · 총 **18장**(보는 강의가 아니라 해보는 워크숍 → 일부러 적게)
> 실제 .pptx 생성은 이번 범위 밖. **이 구성안으로 강사가 직접 제작**(또는 `make-ppt` 스킬 활용).

## A. 제작 방식 추천: **PPT 주력 + 노션 보조 허브**

| 항목 | PowerPoint | 노션 | 구글 슬라이드 | Gamma 등 AI도구 |
|---|---|---|---|---|
| 라이브 데모 전환(앱↔슬라이드) | ◎ 발표자보기·단축키 | △ 탭 전환 어색 | ○ | △ |
| 사내 배포·재사용 | ◎ 사내 표준 | ○ 링크 공유 | ○ | △ |
| 디자인 통제 | ◎ (보유 템플릿 활용) | △ 정형적 | ○ | ○ 자동생성 |
| 시리즈 반복 사용 | ◎ 마스터 슬라이드 | ◎ 링크 허브 | ○ | △ |
| 오프라인·사내망 안정성 | ◎ 인터넷 불필요 | △ 인터넷 필요 | △ | △ |
| 비개발자 친숙도 | ◎ | ○ | ○ | △ |

**추천: PPT 주력 + 노션을 "교육 시리즈 허브"로.**
1. 이 세미나는 **라이브 데모가 핵심**(`시작.bat`→앱, Claude Code 전환). PPT 발표자보기 + Alt+Tab이 사내망/오프라인에서 가장 안정적.
2. 강사 보유 `make-ppt`·`make-ppt-townhall`·`make-ppt-interx` 스킬로 **시리즈 톤 통일·재사용**.
3. **노션은 발표용이 아니라 "자료 허브"**: 세션 후 PRD 템플릿·CONTEXT 예시·실습 가이드·후속 Q&A를 한 페이지에 모아 링크 공유. (발표=PPT / 배포·축적=노션)
4. Gamma류는 초안엔 좋지만 사내·라이브데모·보안 환경엔 통제·안정성이 부족 → 보조.

> 실행 팁: 아래 슬라이드 구성안을 `make-ppt` 스킬에 넘기면 디자인 통일된 PPT 초안을 바로 생성 가능.

## B. 슬라이드별 구성안 (18장)

## C. 시각화 제안 (공통 비주얼 가이드)

1. **워크플로우 도식 (S10 메인)** — 가로 5단계: 업무 → PRD·맥락 → AI(파이썬·블록·스트림릿) → 웹앱 → 검증. 각 단계에 **사람 아이콘/로봇 아이콘**으로 역할 구분. 검증→PRD 점선 피드백 루프. 색: PRD/맥락 블루(#185FA5), 검증/성공 티얼(#0F6E56), 경고 앰버(#BA7517).
2. **수작업 vs 자동화 (S2, S12)** — 좌우 분할. 좌: 빽빽한 엑셀 + 시계 4~6h + 빨간 실수. 우: 깔끔한 앱 + 30초 + 초록 체크. 하단 띠그래프로 연 60h 절감.
3. **개념 비유 그림 (S4~S9)** — 파이썬=통역사 / 라이브러리=레고 블록4 / 스트림릿="코드 한 줄→화면 요소" 매핑 / 바이브코딩=말풍선→시공팀→집 / PRD=막연 주문 vs 상세 주문서 + 하자점검 돋보기.
4. **실제 자산 스크린샷 (신뢰도↑)** — `requirements.txt` 5줄, `app.py`의 `st.metric`·`st.file_uploader`, `02_PRD.md` 목차, 검증 결과 화면, 5시트 엑셀의 조건부 서식(빨강/노랑/청색). "가짜 예시가 아니라 강사 본인 사례"임을 계속 상기.
5. **데모 백업 비주얼** — S11~13은 반드시 단계별 캡처/짧은 녹화를 슬라이드에 숨겨두기(라이브 실패해도 흐름 유지).
