# 📘 AI로 내 업무 자동화하기 — 세션 설계 개요 (최종 보고)

> 사내 **AI 자동화 교육 시리즈** 중 **1~2주차 1~2시간 세미나** 1회분 설계·제작본
> 워크드 예시: 강사(HR 윤성)가 직접 만든 **HR 월간 근태현황 분석 자동화 도구**
> 작성: 2026-06-01

---

## 1. 이 세션이 답하는 질문

> "결국 기존 워크플로우에서 **어떤 부분을, 어떻게 AI로 자동화할까?**"

이 세션은 그 질문을 **구조화·템플릿화**해서, 비개발자 부서원이 스스로
**① 내 워크플로우를 그리고 → ② 자동화 지점을 찾고 → ③ PRD를 써서 → ④ AI에게 시키고 → ⑤ 검증**하는 흐름을 직접 체험하게 합니다.

핵심 철학: **"코딩을 가르치지 않는다. AI에게 일을 제대로 시키는 사고방식을 심는다."**

---

## 2. 대상 · 형식 · 목표

| 항목 | 내용 |
|---|---|
| 대상 | 비개발자 직장인 (엑셀·스프레드시트로 데이터 취합·관리하는 실무자), AI 가벼운 사용 경험, 코딩 경험 거의 없음 |
| 형식 | 멀티-주차 시리즈 중 **1회 세미나 (90분 기본 / 120분 확장)**, 워크숍형 |
| 실습 도구 | Claude Code / Cursor (라이브 데모 + 미니 체험) |
| 슬로건 | "코딩을 배우지 말고, **AI에게 일을 제대로 시키는 법**을 배운다." |

**학습성과 (끝나면 할 수 있는 것)**
1. 내 반복 업무를 워크플로우(입력→처리→출력)로 그린다
2. 어디를 AI로 자동화할지 짚어낸다 (반복+규칙+시간 / 사람 판단은 분리)
3. 미니 PRD(요구사항 1장)를 쓴다
4. "맥락 → PRD → 시킴 → 검증 → 개선" 한 사이클을 이해한다

**핵심 메시지 5개** (세션 내내 후렴구처럼 반복)
- (a) AI에게 **맥락(context)을 충분히** 줘라
- (b) "그냥 만들어줘" ❌ → **PRD/명세 먼저** ✅
- (c) AI 산출물은 **검증**하라 (정답과 대조)
- (d) **반복 개선**(iterate)한다
- (e) 한 번에 완벽 안 됨 — **대화로 다듬는다**

## 3. 워크드 예시 (세션 공통 언어)

**HR 월간 근태정리 자동화** — 강사 본인의 실제 프로젝트라 신뢰도·생생함이 핵심 자산.

- **Before**: 출퇴근 시스템 2개(원티드스페이스 + FLEX) 데이터를 수작업 엑셀로 병합 → 43명 소정시간 대비 초과·야간 계산 → 동명이인·시스템전환자 등 예외 수기 처리 → 검증 → 보고. **월 4~6시간.**
- **전환점**: "근태 프로그램 만들어줘"가 아니라 **문서 두 장**을 먼저 씀
  - `01_CONTEXT.md` = 비즈니스 맥락 (인원·시스템·예외 케이스)
  - `02_PRD.md` = 제품 요구사항 (목표·입력·출력·검증)
- **AI에게 시킴**: Claude Code가 PRD 기반으로 **Streamlit 웹앱** 생성
- **검증**: 결과 엑셀을 수기 최종본과 **100% 대조** 후 채택
- **After**: 월 4~6시간 → **수십 분**. 양식 일관·검증 자동·구글시트 공유·배포까지.

> 실증 자료(시연용): `02_PRD.md`, `01_CONTEXT.md`, `CLAUDE.md`, `app.py`, `requirements.txt`, `lib/auth.py`, `lib/config_store.py`, `시작.bat`

## 8. 시리즈 로드맵 (이 세션의 위치)

- **1~2주차 (이 세션)**: 사고방식 — 워크플로우 그리기 → 자동화 포인트 → PRD → AI에게 → 검증
- **다음 주차(예시)**: 실습 환경 세팅 + 내 PRD로 실제 스캐폴딩 + 검증/개선 사이클
- **이후(예시)**: 사내망/클라우드 배포, 구글시트·슬랙 연동, 부서별 미니 프로젝트 발표
