# 📄 미니 PRD 템플릿 + 부서별 완성 예시

> PRD = 요구사항 명세 = **AI에게 줄 주문서**. 이 한 장을 채워서 Claude Code/Cursor에 그대로 붙여넣으면 됩니다.
> 코딩 지식 불필요 — **여러분 머릿속 규칙을 글로 꺼내는 것**이 전부입니다.

---

## ✏️ 빈 템플릿 (복사해서 사용)

```markdown
# 미니 PRD: (도구 이름)

작성자: ______    날짜: ______

## 1. 문제 / 목적
- 지금 무엇이 불편한가:
- 이루고 싶은 것(한 문장):

## 2. 사용자
- 누가 쓰나:
- 언제 / 얼마나 자주:

## 3. 입력 (무엇이 들어오나)
- 파일/데이터 종류:
- 형태(컬럼·시트·형식):
- 어디서 가져오나:

## 4. 출력 (무엇이 나와야 하나)
- 결과물 형태(엑셀/시트/문서):
- 꼭 포함될 항목:

## 5. 핵심 규칙 · 계산 (AI가 따라야 할 규칙)
- 규칙 1:
- 규칙 2:
- 계산식/기준:

## 6. 엣지케이스 (예외 — 이런 경우엔 이렇게)
-

## 7. 하지 않을 것 (Non-goals)
-

## 8. 성공 기준 (이러면 성공 / 무엇과 대조해 검증?)
-
```

> 💡 **5번(규칙)·6번(예외)이 PRD의 심장.** 여기가 구체적일수록 결과가 정확합니다.
> 모르는 칸은 "모름/확인필요"라 적어도 됩니다 — 그게 AI에게 물어볼 거리가 됩니다.

## ✅ 완성 예시 ① — HR: 월간 근태현황 분석

```markdown
# 미니 PRD: 월간 근태현황 분석 자동화 도구
작성자: HR 담당   날짜: 2026-06-01

## 1. 문제 / 목적
- 불편: 출퇴근 시스템 2곳 데이터를 손으로 병합·계산하느라 매월 4~6시간, 실수 위험.
- 목표: 두 시스템 엑셀을 올리면, 직원별 근태 집계·검증 결과를 자동으로 받는다.

## 2. 사용자
- HR 근태 담당자 / 매월 1회, 월초 마감 때

## 3. 입력
- 원티드스페이스 엑셀, FLEX 엑셀 (각 1개)
- 직원명·사번·날짜·출근·퇴근 컬럼. 시스템마다 컬럼명·형식 다름.
- 각 시스템에서 월별 다운로드

## 4. 출력
- 엑셀 1개(5시트) + 구성원 공유용 구글시트
- 포함: ①통합 원본 ②개인별 집계(소정/초과/야간) ③검증(이상치) ④요약 ⑤확인필요 목록

## 5. 핵심 규칙 · 계산
- 사번 기준 병합(이름 표기 차이 보정)
- 소정시간 대비 초과/야간 구분 계산 (야간 22:00~06:00)
- 반올림 규칙은 사내 규정에 따름

## 6. 엣지케이스
- 동명이인·시스템 전환자·소정시간 불일치 → "확인필요" 시트로 빼서 사람이 검토

## 7. 하지 않을 것
- 급여/지급액 계산 안 함. 근태 규정 자체를 바꾸지 않음. 시스템 자동 로그인 범위 밖.

## 8. 성공 기준
- 손계산 결과와 자동 결과가 일치 + 이상치가 명확히 표시 + 전체 30분 이내면 성공.
```

## ✅ 완성 예시 ② — 영업: 주간 매출 취합

```markdown
# 미니 PRD: 주간 매출 취합 도구
작성자: 영업 담당   날짜: ______

## 1. 문제 / 목적
- 불편: 매주 3개 지점 매출 엑셀을 손으로 합치는 데 1.5시간, 지점마다 양식이 다름.
- 목표: 지점 엑셀 3개를 올리면 통합 요약표를 자동으로 받는다.

## 2. 사용자
- 영업관리 담당자 / 매주 월요일 오전

## 3. 입력
- 지점별 매출 엑셀 3개 (.xlsx)
- 공통 컬럼: 날짜, 품목, 수량, 금액 (지점마다 컬럼 순서·이름 조금 다름)
- 각 지점 담당이 메일로 보냄

## 4. 출력
- 엑셀 1개: '지점요약'(지점별 총매출, 전주 대비 증감) + '품목별'(품목 합계)
- 합계 행은 굵게

## 5. 핵심 규칙 · 계산
- 3개 파일을 하나로 합치고 날짜 기준 정렬
- 품목별·지점별 금액 소계
- 마이너스(환불) 금액은 합계에서 차감

## 6. 엣지케이스
- 지점명 표기 차이('강남점'='강남') → 같은 곳 취급
- 날짜 형식이 파일마다 다름(2026-01-01 / 1/1) → 자동 인식
- 빈 행 무시

## 7. 하지 않을 것
- 매출 예측·목표 대비 분석은 이번 범위 밖(단순 취합·요약만)

## 8. 성공 기준
- 지난주 손으로 만든 결과의 지점별 총합과 일치하면 통과
```

## ✅ 완성 예시 ③ — 마케팅: 캠페인 성과 리포트

```markdown
# 미니 PRD: 주간 광고 성과 리포트 도구
작성자: 마케팅 담당   날짜: ______

## 1. 문제 / 목적
- 불편: 채널 3곳(네이버·메타·구글) 데이터를 채널마다 다른 양식으로 정리하느라 매주 2시간.
- 목표: 채널별 광고 데이터를 올리면 통합 성과표를 자동으로 받는다.

## 2. 사용자
- 퍼포먼스 마케터 / 매주 1회

## 3. 입력
- 채널별 광고 데이터 엑셀/CSV 3개
- 공통 개념: 캠페인명, 비용, 노출, 클릭, 전환, 매출 (채널마다 컬럼명 다름)
- 각 광고 콘솔에서 내려받음

## 4. 출력
- 엑셀 1개: '캠페인별 요약'(비용·전환·ROAS) + '채널별 합계'
- ROAS = 매출 / 비용, CPA = 비용 / 전환

## 5. 핵심 규칙 · 계산
- 채널별 다른 컬럼명을 공통 항목으로 매핑
- 캠페인 단위로 합산, ROAS·CPA 자동 계산
- 비용 0인 행은 ROAS 계산 제외(0으로 표시)

## 6. 엣지케이스
- 같은 캠페인이 채널마다 다른 이름 → 매핑 표로 통일
- 전환·매출 데이터 누락 시 빈칸이 아니라 0 처리

## 7. 하지 않을 것
- 인사이트/전략 코멘트는 사람이 작성(AI는 수치 표까지만)

## 8. 성공 기준
- 콘솔 원본 합계와 통합표 비용·전환 총합이 일치하면 통과
```

## ✅ 완성 예시 ④ — 재무: 부서별 경비 정산 취합

```markdown
# 미니 PRD: 부서별 경비 정산 취합 도구
작성자: 재무 담당   날짜: ______

## 1. 문제 / 목적
- 불편: 매월 부서별 경비 엑셀을 계정과목별로 분류·합산·점검하느라 3시간.
- 목표: 부서별 경비 엑셀을 올리면 계정별 결산표와 점검 리스트를 자동으로 받는다.

## 2. 사용자
- 재무 정산 담당자 / 매월 결산 시

## 3. 입력
- 부서별 경비 엑셀 여러 개
- 공통 컬럼: 일자, 부서, 계정과목, 금액, 증빙유무, 사용내역
- 공유 드라이브/메일로 수합

## 4. 출력
- 엑셀 1개: '부서×계정 합계표' + '점검 리스트'(한도 초과·증빙 누락)
- 합계 행 굵게

## 5. 핵심 규칙 · 계산
- 계정과목 표기 통일(예: '식대'='중식대') 후 부서·계정별 합산
- 부서별 한도 대비 초과 여부 표시

## 6. 엣지케이스
- 증빙 '무' 또는 빈칸 → 점검 리스트에 표시(자동 삭제 X)
- 한도 초과는 자동 차감하지 말고 "확인필요"로만 표시(판단은 사람)

## 7. 하지 않을 것
- 전표 자동 생성·ERP 입력은 범위 밖(취합·점검까지만)

## 8. 성공 기준
- 부서별 총합이 원본 엑셀 합과 일치 + 증빙 누락 건이 빠짐없이 잡히면 통과
```

## 🎯 PRD를 쓰고 나면 — AI에게 이렇게 시작하세요

> "나는 (부서/역할)이야. (반복 업무)를 자동화하려고 해. 아래 PRD를 첨부할게.
> 우선 **입력 파일을 읽어서 합치는 1단계까지만** 만들어줘. 내가 확인하면 계산으로 넘어가자.
> 다 되면 **결과를 검증할 수 있는 표/시트**도 같이 만들어줘."
>
> (그리고 위 PRD 전체를 붙여넣기)

**기억할 것**: ① 맥락부터 ② 단계로 쪼개서 ③ PRD 그대로 ④ **반드시 검증** ⑤ 대화로 개선.
