# 🎤 진행자 가이드 — "AI로 내 업무 자동화하기"

> 형식: 90분 기본 / 120분 확장 · 비개발자 워크숍
> **진행자 마인드셋**: 목표는 "코딩 교육"이 아니라 **"AI에게 일을 제대로 시키는 사고방식"** 심기. 코드 한 줄 못 짜도 됨. 핵심은 **PRD를 쓰는 머리**.

---

## 1. 세션 목표 & 학습성과

이 세션이 끝나면 참가자는…
1. 자기 업무를 **워크플로우(입력→처리→출력)로 그릴 수 있다**
2. **어디를 AI/자동화로 바꿀지 짚어낼 수 있다** (반복·규칙·시간 / 사람 판단은 분리)
3. **미니 PRD(요구사항 1장)를 쓸 수 있다**
4. **"맥락 → PRD → 시킴 → 검증 → 개선"** 사이클을 체험으로 안다

### 블록 0 — 오프닝 / 훅 (7' / +2h 10')
- **목적**: "나도 할 수 있겠다" 기대치 + 코딩 공포 제거
- **대본 요지**:
  - "오늘 끝나면 여러분은 코드를 짤 줄 알게 되지 **않습니다.** 대신 **AI에게 내 일을 시키는 법**을 알게 됩니다."
  - 훅: "제가 매달 4~6시간 걸리던 일을 **수십 분으로** 줄였습니다. 코드는 거의 안 썼고, **문서 두 장**을 썼어요." (뒤 스토리 예고)
  - 손들기 폴: "엑셀에서 매달/매주 똑같은 작업 반복하시는 분?" → "그게 오늘의 재료입니다."
- **참가자**: 손들기 + 옆 사람과 "내가 제일 귀찮은 반복 업무" 한 마디(10초)
- **(+2h)**: 2~3명 마이크 → 화이트보드에 적어둠(나중 실습 예시로 재활용)

### 블록 1 — 개념 미니강의 (10' / +2h 13')
- **목적**: 데모/실습 중 나올 단어에 겁먹지 않게 비유로 깔기. **깊게 X.**
- **용어 카드 (각 1~2분, 비유 중심)** — 상세는 `03_개념설명_비개발자용.md`

| 용어 | 한 줄 대본 |
|---|---|
| 파이썬 | "컴퓨터에 일 시키는 언어, 제일 쉬운 축. **우리가 말 안 하고 AI가 대신 씁니다.**" |
| 라이브러리 | "남이 만든 기능 묶음 = 라면스프. 엑셀스프(openpyxl)·표스프(pandas)·웹앱스프(streamlit)." |
| 스트림릿 | "파이썬으로 **버튼·업로드·표 있는 웹 화면**을 쉽게. 우리가 본 그 근태 도구 화면이 이거." |
| 바이브코딩 | "자연어로 AI에게 코딩 시키기. **단, '느낌적으로 만들어줘'가 아니라 PRD로 방향을 줘야** 함." |
| (+2h) PRD | "요구사항 명세 = **주방에 넘기는 주문서.** 무엇을·왜·입력·출력을 적은 한 장." |

- **팁**: 여기서 시간 끌면 전체가 무너짐. **"지금 다 이해 못 해도 됩니다. 뒤에서 실제로 보면 와닿아요"** 하고 빠르게.

### 블록 2 — 워크드 예시 스토리 (12' / +2h 15')
- **목적**: 추상 개념을 진짜 사례 하나에 묶고, 세션 내내 공통 언어로 재사용
- **스토리 순서 (슬라이드 4~5장)**:
  1. **Before(Pain)**: "HR팀이 매달 파견·계약직 **43명** 근태 정리. 시스템이 **두 개**(원티드+FLEX). 손으로 붙여넣고, 사람마다 초과·야간 계산하고, **예외를 일일이** 처리하고, 검증도 손으로. **매달 4~6시간.**"
  2. **전환점**: "제가 한 건 '프로그램 만들어줘'가 **아니라** 문서 두 장이었어요." → `01_CONTEXT.md`(맥락) + `02_PRD.md`(요구사항)
  3. **AI에게 시킴**: "그 PRD를 Claude Code에 줬더니 **Streamlit 웹앱**을 만들어줬어요."
  4. **검증**: "**그대로 안 믿었습니다.** 결과 엑셀을 제 수기 최종본과 **100% 대조**, 틀린 건 대화로 고침."
  5. **After**: "월 4~6시간 → **수십 분.** 양식 일관, 한도 위반 자동 감지, 구글시트 공유·배포까지."
- **핵심 교훈 5개 화면에 박기**: (a)맥락 (b)PRD 먼저 (c)검증 (d)반복개선 (e)한번에 완벽X
- **팁**: 숫자를 구체적으로(신뢰↑). 단, **실명·사번은 익명/블러.**
- **(+2h)**: 실제 `02_PRD.md`의 "User Story" 한 토막 낭독 — "이렇게 말로 쓰면 됩니다, 코드 없죠?"

### 블록 3 — 실습1: 내 워크플로우 그리기 (13' / +2h 18')
- **목적**: 학습성과 (1). 업무를 "입력→처리→출력"으로 분해 (**워크시트1**)
- **진행**: "거창한 거 말고 **매주/매달 반복하는 엑셀 작업 하나**만." → 칠판에 근태 예시 1줄 시범 → 8분 작성 → 진행자 순회 코칭
- **팁**: 비개발자는 "처리"를 뭉뚱그림("그냥 정리해요"). **"손을 몇 번 움직이는지" 동작 단위로 쪼개라**고 유도. 쪼갠 단계 = 자동화 후보.
- **(+2h)**: 페어로 서로 워크플로우 설명(각 2분) → 빠진 단계 드러남

### 블록 4 — 실습2: 자동화 포인트 찾기 (10' / +2h 13')
- **목적**: 학습성과 (2). AI로 넘길 지점 식별 (**워크시트2**)
- **진행**: 판별 기준 3개 — ① 반복적인가 ② 규칙이 있는가 ③ 시간이 오래/자주 걸리나. "반대로 **그때그때 사람 판단**이 필요한 건 자동화 대상 아님. AI는 **규칙 있는 노가다**를 대신."
- **활동**: 워크시트1 단계마다 ✅(자동화) / ✋(사람 판단) / ⭐(제일 아픈 한 곳) 표시
- **오해 잡기**: "AI가 내 판단까지 대신?" → "아니요. 규칙대로 빠르게. 판단·검증은 사람. 그래서 뒤에 검증이 있는 거예요."

### 블록 5 — 실습3: 미니 PRD 작성 (15' / +2h 20') ★클라이맥스
- **목적**: 학습성과 (3). ⭐ 후보 하나를 AI가 알아듣는 명세로 (**워크시트3 / `05_예제_미니PRD_템플릿.md`**)
- **진행**: "'매출 정리 좀 해줘' 하면 엉뚱한 게 나와요. **주문서**를 써야 합니다." → 6칸 템플릿을 근태 예시로 한 줄씩 시범 → 10분 작성
- **팁**: ④처리 규칙·⑥예외 칸에서 다들 막힘. **"머릿속 암묵적 규칙을 글로 꺼내는 게 PRD의 전부"** 안내. 한 칸이라도 채우면 성공.
- **(+2h)**: PRD 옆 사람과 교환 → "이것만 보고 내가 만들 수 있겠나?" 피드백(2분)

### 블록 6 — 라이브 데모: PRD를 AI에게 (15' / +2h 20')
- **목적**: "PRD → AI → 결과물"이 진짜 작동함을 눈으로. **안전하게, 실패 없이.** (상세는 4절)
- **(+2h)**: 참가자 PRD 1개 자원받아 즉석 데모

### 블록 7 — 마무리 (8' / +2h 11')
- **대본**: 오늘 한 줄 반복 — "**워크플로우 그리기 → 자동화 포인트 → PRD → AI에게 → 검증.**" + 핵심 교훈 5개 재노출
- **숙제**: "오늘 쓴 미니 PRD를 다듬어 오세요. 다음 주엔 **이걸 실제 동작하는 도구**로 만듭니다."
- **회고**: 30초 — "오늘 가장 의외였던 것" 한 줄

## 3. 진행자 팁

### 비개발자 대상 주의점
- **코드 화면을 오래 비추지 마세요.** "이건 AI가 쓴 거, 우리가 읽을 필요 없어요. **우리가 본 건 PRD뿐**" 반복.
- 용어 쓸 때마다 **비유 재호출**.
- **결과물(웹 화면) 먼저, 코드는 나중에(또는 안 보여줌).** 비개발자는 "버튼 누르니 엑셀 나옴"에 반응.
- **개인정보**: 근태 예시 실명·사번·소정시간 익명/블러. 화면 공유 전 점검.

### 자주 나오는 질문 / 오해와 답변

| 질문/오해 | 답변 요지 |
|---|---|
| "코딩 배워야 쓰는 거 아니에요?" | "아니요. 우리가 쓴 건 **PRD(문서)**. 코드는 AI가. 단 PRD를 잘 써야 결과가 좋아요." |
| "AI가 한 번에 완벽하게?" | "절대 아닙니다. 근태 도구도 **여러 번 대화로 고쳤어요.** AI는 초안을 빨리 줄 뿐." |
| "AI 결과 그냥 믿어도?" | "**절대 안 됩니다.** 정답(수기 결과)과 대조 검증 필수. 그래서 PRD에 '검증' 칸이 있어요." |
| "데이터 외부로 새는 거 아닌가?" | "보안 중요. 근태 도구도 **비밀번호 잠금 + 회사 정책에 맞는 배포**로 운영." |
| "우리 업무는 너무 복잡해서…" | "근태도 시스템 2개·예외 10개로 복잡했어요. **복잡할수록 PRD로 쪼개면** 됩니다." |
| "엑셀 매크로랑 뭐가 달라요?" | "매크로는 정해진 동작만. AI는 **규칙을 말로 설명**하면 코드로 바꿔주고, 새 요구도 대화로 추가." |

### 시간 부족 시 줄이는 법 (우선순위)
1. 블록6(라이브 데모) → **3분 녹화 영상으로 대체** (-10분)
2. 블록4를 블록3에 합치기(그리면서 ✅/✋ 마킹) (-7분)
3. 블록1 용어 4개→2개(라이브러리·바이브코딩만), 나머지 유인물 (-4분)
4. (+2h 확장 요소 전부 제거) → 90분 수렴
- **절대 빼지 말 것**: 블록5(미니 PRD 작성). 빠지면 "구경"으로 끝남.

### 분위기 / 참여 유도
- 블록마다 **손들기 폴 1개**(리듬). **"여러분 업무로 치면?"**을 후렴구로.
- 막히는 실습엔 **진행자가 자기 PRD를 큰 소리로 채우며 시범**(빈 화면 공포 제거).
- 발표 강요 X. **"정답 없어요, 초안이에요"** 반복.

## 4. 라이브 데모 시나리오 (Claude Code / Cursor)

> 원칙: **실패하지 않게, 짧게, 결과가 눈에 보이게.** 설치·네트워크 의존 단계는 미리 끝내두고, 현장은 "PRD 넣기 → 생성 → 실행"만.

### 사전 준비 (전날까지)
- 빈 데모 폴더(`데모_매출취합`), 인터넷/모델 접속 검증
- **데모용 PRD 텍스트**를 클립보드/메모장에 대기
- **백업**: 전 과정 3분 녹화 + 완성된 근태 도구 `시작.bat` 즉시 실행 대비
- Streamlit 포트(8501) 브라우저 미리 한 번 워밍업

### 데모 순서 (약 12~15분)
1. (30초) "지금부터 코드 한 줄도 안 칩니다. **PRD만 붙여넣을게요.**"
2. (2분) Claude Code 채팅에 아래 데모용 PRD 붙이고 한 줄 지시:
   > "위 PRD대로 Streamlit 웹앱 **뼈대**를 만들어줘. 파일 올리면 합쳐서 요약 엑셀 내려받는 화면. 완벽한 로직 말고 **화면 뜨고 버튼 보이는 수준**까지만."
3. (3~4분) AI가 파일 만드는 과정 관전 + 내레이션: "지금 `app.py`, `requirements.txt`를 만들고 있어요. 그 라이브러리(streamlit, pandas)를 쓰네요. **우리는 읽을 필요 없습니다.**"
4. (2분) 실행: `streamlit run app.py`(또는 `시작.bat`) → 브라우저에 **업로드 박스+버튼** 화면. "PRD 한 장으로 여기까지."
5. (2분) 반복 개선 시연: "업로드 박스 위에 '주간 매출 취합 도구' 제목과 설명 한 줄 넣어줘" → 바뀌는 화면. "**이게 iterate.**"
6. (1~2분) 검증 메시지로 마무리: "이건 아직 뼈대예요. 실제로 쓰려면 **손 정답과 대조 검증** 필수. 근태 도구는 정답 엑셀과 100% 맞춘 뒤 썼어요."

### 데모용 PRD 예시 (그대로 붙여넣기)

```markdown
# 미니 PRD: 주간 매출 취합 도구

## 무엇을
3개 지점의 주간 매출 엑셀을 업로드하면, 하나로 합쳐
'지점요약'·'품목별' 시트가 있는 엑셀로 내려받는 웹 도구.

## 왜
지금은 매주 약 1.5시간, 지점마다 양식이 달라 손으로 맞춤.

## 입력
- 지점별 엑셀 3개 (.xlsx), 공통 컬럼: 날짜, 품목, 수량, 금액

## 처리 규칙
- 3개 합치기 / 날짜 정렬 / 품목별 소계
- 마이너스(환불) 행은 합계에서 차감 / 빈 행 무시

## 출력
- 엑셀 1개: '지점요약'(지점별 총매출) + '품목별'(합계, 맨 아래 합계행 굵게)

## 예외/주의
- 지점명 '강남점'='강남' 같은 곳 취급 / 날짜 형식 달라도 자동 인식

## 검증
- 지난주 손으로 만든 결과 총합계와 일치하면 통과
```

> 안전장치: 5번에서 응답이 길어지면 끊고 백업 영상으로. 청중은 "한 줄 추가하니 바뀐다"만 보면 충분. **코드의 완벽함이 아니라 '흐름'을 보여주는 게 목적.**

## 6. 한 줄 운영 원칙
> 비개발자에게 코딩을 가르치지 마세요. **(1) 자기 일을 그리고 → (2) 자동화할 곳을 짚고 → (3) PRD 한 장을 손으로 쓰게 하는 것**, 이 셋만 확실히 시키면 성공입니다. 근태정리 사례는 "그게 진짜 통한다"는 살아있는 증거, 데모는 "PRD 한 장으로 화면이 뜬다" 한 방이면 충분합니다.
